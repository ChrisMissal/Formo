Why Use Formo?

Rather than using strings all over the place to access your configuration settings, Formo gives you a bit more flexibility and advantages:

Free casting
Using configuration directly gives you back strings, Formo will let you cast these values into their expected types.
Fallback Values
By calling a setting as a method, you can supply arguments to fallback on if the setting isn't found or is null.
Extensible
When using Formo, you can provide your own custom type providers to turn your settings into any type you'd like.

Basic Use

There's a few things you need to know about basic usage. After you create your Configuration as a dynamic object, simply invoke the setting key as a property or a method like so:

The settings

<appSettings>
<add key="RetryAttempts" value="5" />
<add key="ApplicationBuildDate" value="11/4/1999 6:23 AM" />
</appSettings>

Formo.Configuration

dynamic configuration = new Formo.Configuration();

Getting Values

var retries = configuration.RetryAttempts; // returns "5"

// or better yet

var retries = configuration.RetryAttempts<int>(); // returns 5 as an int

Fallback to Defaults

Another helpful feature that Formo gives you is the ability to provide fallback values if your setting doesn't exist.

When Defaults are Provided

var email = configuration.FromEmail<string>("noreply@pandas.com");

Since the setting "FromEmail" doesn't exist in our settings file, the argument supplied will server as the return value.

Multiple Fallbacks

If your setting can be optional on various levels, but required at some point, you can provide several arguments to a method call to try to obtain the value you need.

var userBackground = GetUserBackground();
var background = configuration.ImageUrl(userBackground, "gradient.png");

In this example, Formo will first check the settings file, then, if not found, will use the "userBackground" value if not null, finally falling back on "gradient.jpg" if "userBackground" is null. The method takes in params object[] so you can pass in as many as necessary.
